# Raft AI Workload Estimator

Профессиональный инструмент для оценки трудозатрат и стоимости AI-проектов, разработанный для компании **Raft**.

## 🚀 О проекте

Этот калькулятор позволяет быстро декомпозировать сложные IT-проекты (с фокусом на AI/LLM), учитывать специфические риски и формировать детализированное коммерческое предложение.

### Ключевые возможности:
- **Семантический поиск**: Умный поиск по базе из 110+ типовых работ (Infrastructure, OCR, RAG, Agents, Compliance и др.).
- **Гибкие множители**: Учет зрелости команды, качества данных и готовности моделей.
- **Динамическая архитектура**: Автоматическая визуализация структуры системы через Mermaid.js.
- **Риск-менеджмент (DoR)**: Снижение буфера рисков при наличии четких требований (Definition of Ready).
- **Экспорт в Excel**: Генерация детальной сметы с распределением по ролям.
- **Интеграция с LLM**: Помощь AI (Gemini или Ollama) в подборе работ.

## 🛠 Технический стек

- **Frontend**: React 18, Vite, Tailwind CSS.
- **Анимации**: Framer Motion.
- **Графики и диаграммы**: Mermaid.js, Recharts, Lucide Icons.
- **AI**: Google Gemini API или локальный Ollama.

## 📈 Логика расчета

Итоговая стоимость и сроки рассчитываются по формуле:
`Total = BaseHours * EnvironmentMultiplier * RiskMultiplier`

- **BaseHours**: Сумма часов выбранных задач (с учетом коэффициента количества). По умолчанию 1 задача = 100% базовых часов.
- **EnvironmentMultiplier**: Произведение множителей (Сложность * Команда * Данные * Модель). Это "внешний" фон проекта.
- **RiskMultiplier**: Базовый риск **1.80 (80%)**, который заложен на неопределенность. Каждый отмеченный пункт в чек-листе **DoR (Definition of Ready)** снижает этот коэффициент, так как уменьшает риски.

## 📖 Подробное руководство пользователя

Процесс нормирования в калькуляторе состоит из 5 ключевых этапов:

### 1. Настройка окружения (Левая панель)
*   **Глобальные факторы**:
    *   *Сложность системы*: Влияет на накладные расходы архитектуры.
    *   *Зрелость команды*: Senior-команда работает эффективнее (множитель < 1), Junior — медленнее (> 1).
    *   *Качество данных*: Если данные "грязные", DS-задачи могут занять в 1.8 раза больше времени.
*   **Чек-лист DoR (Definition of Ready)**: 
    *   **Почему это важно?** В AI-проектах самая большая проблема — неопределенность. Если у вас нет ТЗ или данных, проект "рискованный". 
    *   **Как это работает?** По умолчанию включен буфер 1.8х (Добавочный риск). Когда вы отмечаете "Данные доступны" или "Цели зафиксированы", этот буфер уменьшается. Это мотивирует заказчика предоставлять четкие требования.

### 2. Подбор и декомпозиция (Вкладки Catalog / Active)
*   **Умный поиск**: Введите "RAG" или "Postgres", и AI предложит подходящие задачи из базы (110+ нормативов).
*   **План проекта**: Здесь вы можете менять количество (например, 3 интеграции вместо 1) и менять исполнителя (Role). Стоимость пересчитается мгновенно на основе рейтов ролей.

### 3. Работа с архитектурой (Вкладка Diagram)
*   Сервис автоматически строит C4-диаграмму верхнего уровня на основе выбранных вами работ. Это помогает визуализировать "состав" системы для КП.

### 4. Ресурсное планирование
*   **Параллельные потоки**: Укажите, сколько стримов разработки будет идти одновременно. Это не меняет часы (объем), но уменьшает календарный срок (Duration).

### 5. Финализация и Экспорт
*   **Summary**: Итоговые графики распределения трудозатрат по ролям и категориям.
*   **Экспорт КП**: Сохраняет Excel-файл с детальной сметой, которую можно приложить к договору.

## 💻 Локальный запуск

Если вы экспортировали проект из AI Studio:

1. **Установите зависимости**:
   ```bash
   npm install
   ```

2. **Настройте переменные окружения**:
   Создайте файл `.env`.
   ```env
   GEMINI_API_KEY=ваш_ключ
   ```

3. **Запустите**:
   ```bash
   npm run dev
   ```

## 🤖 Поддержка Ollama

Если у вас нет Gemini API ключа, вы можете использовать Ollama:
1. Установите Ollama и запустите модель (`ollama run llama3`).
2. Настройте `OLLAMA_ORIGINS="*"`.
3. Система автоматически переключится на Ollama при отсутствии ключа API.
